## Critical

### C1. Arbitrary code execution via `$(eval ...)`
- **File**: `src/play_launch_parser/.../substitution/eval.rs:152-198`
- Python's `eval()` is called on expressions from launch files with no
  sandboxing. A malicious launch file can execute arbitrary Python:
  `$(eval __import__('os').system('rm -rf /'))`.
- **Fix**: Pass restricted globals to `py.eval()`:
  `{"__builtins__": {}}`. This blocks `__import__`, `exec`, `open` etc.
  while allowing arithmetic and string operations.

### C2. Arbitrary shell execution via `$(command ...)`
- **File**: `src/play_launch_parser/.../substitution/types.rs:202-251`
- `$(command ...)` runs arbitrary bash commands. This is a standard ROS 2
  feature — the spec requires it. Cannot be removed without breaking
  compatibility.
- **Fix**: Add `--allow-command-substitution` flag (default off for
  `dump`, on for `launch`). Document the risk. Add a timeout (see L3).

### C3. Arbitrary Python execution via `.launch.py` files
- **File**: `src/play_launch_parser/.../python/executor.rs:110-153`
- Python launch files are executed with `py.run()`. This is inherent to
  ROS 2 — Python launch files are programs by design.
- **Fix**: Document the risk. Consider `--no-python` flag for analysis-only
  mode that refuses Python launch files.

## High

### H1. SSE broadcast blocks on slow subscriber
- **File**: `src/play_launch/src/web/broadcaster.rs:52`
- `tx.send(event).await` blocks on full channels. One slow SSE client
  stalls ALL state event delivery and ALL actor state transitions.
- **Fix**: Replace with `tx.try_send()` — drop events for lagging
  subscribers. Or use `tokio::sync::broadcast` channel.

### H2. Integer overflow in SPSC ring buffer allocation
- **File**: `src/spsc_shm/src/lib.rs:122`
- `HEADER_SIZE + capacity * element_size` can overflow with adversarial
  inputs, causing a small mmap followed by out-of-bounds writes.
- **Fix**: Use `checked_mul` + `checked_add` with error return.

### H3. Unsafe raw pointer for thread-local LaunchContext
- **File**: `src/play_launch_parser/.../python/bridge.rs:292-362`
- `CURRENT_LAUNCH_CONTEXT` stores a raw `*mut LaunchContext`. The
  `get_current_launch_context()` is `pub`, allowing cached stale
  pointers. Multiple guards without dropping silently override.
- **Fix**: Make `get_current_launch_context()` private. Add debug
  assertion in `LaunchContextGuard::new()` that panics if already set.

### H4. No include depth limit (stack overflow risk)
- **File**: `src/play_launch_parser/.../traverser/include.rs`
- Recursive include processing has no max depth. Non-circular but deeply
  nested includes cause stack overflow.
- **Fix**: Add `MAX_INCLUDE_DEPTH = 100` check on `include_chain.len()`.

### H5. No file size limit on cached files
- **File**: `src/play_launch_parser/.../file_cache.rs:22-47`
- `read_file_cached()` reads into memory with no size limit. Cache has
  no eviction policy.
- **Fix**: Add max file size (10MB) and max cache entries (1000).

## Medium

### M1. Overly permissive CORS policy
- **File**: `src/play_launch/src/web/mod.rs:114-117`
- `allow_origin(Any)` in production. Any website on the same network can
  make cross-origin requests to start/stop ROS nodes.
- **Fix**: Restrict to bind address origin. Add `--cors-origin` flag.

### M2. SSE connection resource exhaustion
- **File**: `src/play_launch/src/web/broadcaster.rs`
- No limit on concurrent SSE subscribers. Unbounded memory growth per
  connection.
- **Fix**: Cap at 50 subscribers. Return 503 when exceeded.

### M3. Process tree scan per-recursion level
- **File**: `src/play_launch/src/process/tree.rs:27-71`
- `find_all_descendants` creates `System::new()` + `refresh_processes()`
  on each recursive call. O(depth) full process table scans.
- **Fix**: Single scan at top level, build parent-children map in memory.

### M4. YAML parser deprecated + potential CVE
- **File**: `src/play_launch_parser/.../traverser/yaml.rs:23`
- Uses `serde_yaml` which is deprecated. Past CVEs for billion-laughs
  via YAML aliases.
- **Fix**: Migrate to `serde_yml` or add content-length limit.

### M5. `#![allow(unsafe_op_in_unsafe_fn)]` crate-wide
- **File**: `src/play_launch_parser/.../lib.rs:6`
- Suppresses useful lint for entire crate, not just PyO3 modules.
- **Fix**: Scope to PyO3 modules only, or upgrade to PyO3 0.23+.

### M6. Global package cache — no eviction
- **File**: `src/play_launch_parser/.../substitution/types.rs:146`
- `PACKAGE_CACHE` is process-lifetime. Stale entries after package
  rebuild never evicted.
- **Fix**: Add TTL or `clear_caches()` API.

### M7. Interception hooks return error on missing runtime
- **File**: `src/play_launch_interception/src/lib.rs:201-328`
- When `librcl.so` can't be resolved, all hooks return `RCL_RET_ERROR`.
  This silently breaks the ROS application instead of being inert.
- **Fix**: Return 0 (success/no-op) for init hooks when runtime is None.

### M8. No path traversal prevention in includes
- **File**: `src/play_launch_parser/.../traverser/include.rs:15-34`
- Include paths from substitutions are used directly. A launch file
  could include `../../../../etc/passwd`.
- **Fix**: Acceptable for local tool. Add `--restrict-paths` flag for
  CI/multi-tenant use.

### M9. Python namespace parity gap
- **File**: `python/play_launch/dump/visitor/include_launch_description.py:24`
- Python scope `ns` uses `ros_namespace` config key while Rust uses
  `current_namespace()`. May differ in edge cases.
- **Status**: Accepted — cross-parser comparison validates parity.

### M10. Container scope-map key mismatch (Phase 30)
- **File**: `src/play_launch/src/commands/replay.rs:593-596`
- **Fixed** during this audit: was using `exec_name`, now uses `c.name`.

## Low

### L1. PID reuse race window
- **File**: `src/play_launch/src/member_actor/regular_node_actor.rs:119-133`
- Brief window between spawn and PID registration.
- **Status**: Acceptable — mitigated by Linux sequential PID allocation.

### L2. Stop/Restart don't wait for child after kill
- **File**: `src/play_launch/src/member_actor/regular_node_actor.rs:388-427`
- `child.kill()` without `child.wait()` creates brief zombie.
- **Fix**: Add `child.wait().await.ok()` after kill.

### L3. No timeout on `$(command ...)` execution
- **File**: `src/play_launch_parser/.../substitution/types.rs:207-213`
- `Command::new("bash").output()` blocks indefinitely.
- **Fix**: Use `wait_timeout` (e.g., 30 seconds).

### L4. `read_last_n_lines` reads entire file
- **File**: `src/play_launch/src/web/sse.rs:181-189`
- Reads whole file then takes last N. OOM on large log files.
- **Fix**: Reverse-seek approach.

### L5. Broad exception swallowing in Python include args
- **File**: `python/play_launch/dump/visitor/include_launch_description.py:37`
- `except Exception: pass` silently swallows resolution errors.
- **Fix**: Log at debug level instead of swallowing.

### L6. `SystemTime::now().unwrap()` in anon substitution
- **File**: `src/play_launch_parser/.../substitution/types.rs:122-125`
- Panics if system clock is before UNIX epoch.
- **Fix**: Use `.unwrap_or_default()`.

### L7. Zombie anchor process on panic
- **File**: `src/play_launch/src/process/pgid.rs:17-44`
- Anchor zombie leaked if task panics before `wait()`.
- **Fix**: Wrap in Drop guard.

### L8. OperationGuard Drop spawns detached task
- **File**: `src/play_launch/src/web/handlers.rs:40-49`
- Detached tokio task may not run during shutdown.
- **Status**: Acceptable — stale entries during shutdown are harmless.

### L9. `ftruncate` size cast truncation
- **File**: `src/spsc_shm/src/lib.rs:131`
- `total_size as off_t` could truncate for huge allocations.
- **Fix**: Add `debug_assert!` or `TryInto`.

### L10. Python private member access
- **File**: `python/play_launch/dump/visitor/node.py:41-89`
- Accesses `_Node__ros_arguments`, `_ExecuteLocal__respawn` etc.
  May break between ROS 2 distributions.
- **Fix**: Document version dependency. Add try/except guards.
